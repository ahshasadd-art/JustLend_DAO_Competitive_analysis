📊 2.1 借贷市场 参与竞品：Aave V3 · Aave Pro (V4) · Compound V3 · SparkLend · Venus Core · Venus Flux · Lista DAO · Morpho

JustLend 现状

  • 完整借贷流程：Supply / Borrow / Repay / Withdraw
  • 资产列表展示 APY、利用率、流动性
  • 资产详情页展示协议参数
  • 抵押开关（是否将资产作为抵押品）
  • 偏弱 进度条已有渐变色（绿→红），风险值数字高风险时变红，但超出安全阈值时缺少动态文字警告及建议操作引导（补多少/还多少）
  • 缺失 无市场分类/资产标签体系
  • 缺失 无批量/多步合并操作能力
  • 缺失 无一键循环杠杆
  • 缺失 无抵押品还款
  • 缺失 无抵押品换仓
  • 缺失 无债务换仓

竞品现状

Aave V3 E-Mode 效率模式 + 隔离模式 E-Mode 支持同组资产更高 LTV（稳定币组最高 93–97%）；隔离模式限制新资产风险蔓延；Supply/Borrow Cap 接近程度界面可见。查看图片
Aave Pro (V4) Swap 套件 + 健康因子颜色预警 未来支持抵押品换仓、债务换仓、抵押品直接还款；交易弹窗实时展示健康因子（黄/红分级警告）；Position Manager 策略授权执行。
Compound V3 极简市场语义 + 批量调用 清晰区分 Base Asset（生息+借款）与 Collateral Asset（仅抵押）；Bulker 批量调用，Supply + Approve 合并为单次签名（合约调用，且需要给 Bulker 前置授权）。查看图片
SparkLend Sky 生态借贷 + USDS 低利率 主稳定币 USDS（DAI 升级版）；Save 与 Borrow 分入口；单页 Dashboard 原地操作。
Venus Core 一键循环杠杆 + 抵押品还款 + E-Mode Boost 一键循环（倍数滑杆 + 实时健康因子预览）；抵押品直接还款；E-Mode 分层提升 LTV；净收益展示。查看图片
Venus Flux Smart Collateral + 内置 Swap + 一键杠杆 Smart Collateral（抵押品同时做 DEX 流动性，赚取双重收益）；站内兑换无需跳转；Multiply 一键杠杆。查看图片
Lista DAO 固定利率借款 + LP 双收益展示 固定利率/固定期限借款（7/14/30 天）；LP 抵押品同时展示抵押价值与做市费率；借款利率上限（30% APR）前端可见。查看图片
Morpho 策展金库透明度 + Bundler 批量操作 每个市场由五参数唯一定义（借款资产、抵押资产、预言机、利率模型、LLTV）；策展金库展示策展人、底层市场分配权重与收益贡献；Bundler 批量交易减少签名次数。

迭代建议

核心差距：缺乏高级操作工具（循环杠杆、换仓、抵押品还款），用户完成复杂策略须多步手动操作，且风险告警仅有视觉提示，无行动引导。

  • 优先风险阈值触发时补充动态文字警告 + 操作引导（"需补 X TRX 恢复安全"，直达补仓入口）
  • 优先抵押品直接还款：还款弹窗内选抵押品 → 对接 SunSwap 单步完成，对风险告急用户价值极高
  • 中期一键循环杠杆：先做向导式多步引导（存款→借款→再存款），中期评估 Router 合并执行
  • 中期抵押品换仓 / 债务换仓
  • 中期借款弹窗增加安全缓冲滑杆（设定目标 Risk Value → 反算可借额度）
  • 评估批量操作（多步合并为单次签名，参考 Compound Bulker）
🔒 2.2 质押模块 参与竞品：Aave V3 · Aave Pro (V4) · Venus Core · Lista DAO · SparkLend（Compound V3 无质押模块，Morpho 无原生质押代币机制）

JustLend 现状

  • sTRX 质押：Stake TRX → sTRX
  • 展示质押 APY、总质押量、提取队列
  • 偏弱 收益来源未拆解（节点奖励 vs 协议激励）
  • 缺失 无质押前收益变化预览
  • 缺失 质押仅展示 APY，无多元权益说明（注：sTRX 质押为 TRX → sTRX 的节点收益机制，与治理投票无关；JustLend 治理由 JST 持有者参与，二者为独立体系）
  • 缺失 无底层资产去向可视化

竞品现状

Aave V3 / Aave Pro (V4) V3 Safety Module：质押 AAVE → stkAAVE，单一资产覆盖全协议风险，出现坏账时 AAVE 持有者承担最高 30% 的 Slashing 损失。Umbrella：升级为多资产质押（aToken 等），每种资产仅保护其对应市场，风险精准挂钩、不再由 AAVE 单独兜底全局。两种模式均在界面内联展示风险说明与收益来源拆解。查看图片
Venus Core 锁仓 XVS 同时获得：① 质押收益 ② 治理投票权 ③ Venus Prime 权益资格，三重权益一页清晰展示。查看图片
Lista DAO 锁仓 LISTA → veLISTA；锁仓时长与权益正比（可视化权重曲线）；涵盖收益加成 / 治理权重 / 协议费分配三类权益。查看图片
SparkLend SPK 质押独立入口；质押可参与治理和收益分配；预挖进度在 Data Hub 透明可查。查看图片

迭代建议

核心差距：质押信息密度低，用户感知不到收益来源与底层资产流向，决策依据不足。

  • 优先收益来源拆解：节点奖励占比 vs 协议激励占比，让用户知道钱从哪来
  • 优先质押输入框实时换算年化收益金额（如 "质押 1,000 TRX → 预计年收益 ≈ 62 TRX / $19.6"），APY 百分比转化为具体金额
  • 中期展示 sTRX 底层节点分配方式与各节点收益明细，提升透明度

  • 优先投票权委托功能，降低散户参与门槛
  • 中期"与我相关提案"筛选（按用户持仓资产自动匹配）

迭代建议

核心差距：风险工具分散三处、无主动推送，告警触发后用户不知道该做什么。

  • 优先告警触发时补充动态文字预警（"风险值已超安全线"）+ 操作引导（需补 X TRX / 还 Y USDT，直达入口）
  • 优先Risk Alert / Risk Value / Liquidation Tool 整合为统一安全中心，一个入口管理全部
  • 中期增加邮件 / 站内信主动通知渠道，风险事件离线也能触达
  • 中期关键参数旁（抵押率、借款上限）加 Tooltip 内联说明，减少用户跳文档

3差异功能竞品分析

这一节只保留最有决策价值的信息：竞品怎么做、为什么值得看、JustLend 当前应不应该跟。
Aave V3
本轮不做 协议层能力

E-Mode 效率模式（Efficiency Mode）

  • 竞品做法：把高相关资产分组，提高同组借款空间，并用隔离模式控制新资产风险。
  • 价值：对稳定币互借、LST 循环这类策略场景，资金效率提升最明显。
  • JustLend 建议：当前先做资产分组展示和预期提示，真正的 E-Mode 留到协议升级窗口。
Aave Pro (V4)
版本说明：Aave V4 已于 2026 年 3 月 30 日上线；这里仅保留对 JustLend 最有借鉴价值的前端与交互能力。
可优先借鉴 交易闭环

Swap 功能（抵押品换仓 / 债务换仓 / 意图型限价单）

  • 竞品做法：把抵押品换仓、债务换仓、还款换资产全部收进借贷流程，用户不用跳外部 DEX。
  • 价值：最直接缩短“调仓、还款、救仓”路径，对完成率和风险修复都很关键。
  • JustLend 建议：先做 Repay with Collateral，再扩展 Debt Swap 和 Collateral Swap；限价单放后面。
中期评估 机构向

Position Manager

  • 竞品做法：允许用户授权外部 Manager 代执行调仓、还款和再平衡，并可随时撤销。
  • 价值：更适合机构和高净值用户，不是当前零售体验的第一优先级。
  • JustLend 建议：短期先做 watch-only 多地址视图，中期再评估授权执行机制。
分阶段借鉴 降门槛

批量交易（Multi-call Batch）

  • 竞品做法：把 Approve、Supply、Repay、Withdraw 等多步操作压成一次确认。
  • 价值：明显减少签名和 Energy 消耗，也降低新手中途放弃率。
  • JustLend 建议：短期先做向导式多步流，中期再接 Router 合并执行。
可直接借鉴 零售体验

收益目标规划器（Aave App 核心功能）

  • 竞品做法：用目标金额和时间反推收益曲线，并把收益变化实时可见化。
  • 价值：更适合零售用户理解 sTRX 质押这类长期持有型产品。
  • JustLend 建议：网页端优先在 sTRX 页面落地，先验证体验价值，再考虑 App 化。
Compound V3
近期动态：Compound 正把激励集中到核心市场，说明单纯“撒激励”不再是主战场，产品体验和信息表达的重要性在上升。
不适用 版本迁移场景

版本迁移引导 Banner（V2 → V3）

  • 竞品做法：旧版退场时用强提示 Banner 推动用户迁移，避免仓位长期遗留。
  • 价值：只适合存在长期双版本并行的协议。
  • JustLend 建议：当前历史市场处理方式更直接，这项不构成明显缺口。
差异化机会 低成本优化

市场资产分类标签（JustLend 差异化机会）

  • 竞品现状：各主流竞品均无显式的资产分类标签 UI。Compound V3 通过页面布局区分 Base Asset 与 Collateral Asset（前者为市场主币，后者列于抵押表格），但不贴标签、不支持筛选；Aave V3 有 E-Mode 分组，但也未在资产列表中以标签形式呈现。
  • 价值：Compound 的资产分层逻辑提供了参考，在此基础上增加显式分类标签，成本低，但能持续降低用户理解成本，也方便未来引入更多新资产。
  • JustLend 建议：主动补齐这一缺口，先落地稳定币、TRX 原生、BTC·ETH 映射、历史市场四类标签并支持筛选。

4总结

综合第二、三部分，按优先级与投入产出排序，仅保留 JustLend 当前可落地的事项。
排列原则：优先做高频、见效快、无需改协议的事项；协议层重构和重资产投入不纳入本轮。
优先级 功能名称 参考来源 改动量 紧迫程度 核心价值说明
P0 统一 Dashboard（仓位总控台） Aave Pro 中（前端+数据聚合） ⚡ 极高 把借贷、sTRX、Energy Rental 放到一屏，并提供快捷操作。先解决入口割裂，提升回访和二次操作效率。

P0 失败交易恢复 / Energy 修复流 JustLend 差异化 中（前端+错误码归因+Energy Rental 联动） 🔥 高 交易失败后直接定位原因；Energy 不足时提示缺口并跳转 Energy Rental。重点是减少掉单。

P2 sTRX 底层节点分配权重展示 Morpho · SparkLend 中（前端+数据聚合） 中 展示节点分配、APY 贡献和资金流向，回答用户“钱去了哪里”。
P2 批量操作向导（多步合并） Compound V3 · Aave 中大（前端向导 + 可选 Router） 中 先做多步引导，再评估合并签名，减少确认次数和 Energy 消耗。
P2 Insights / Data Hub 数据看板 SparkLend 中大（数据中台 + 前端） 中 沉淀利用率、清算、sTRX 规模和 Energy 价格趋势，提升对外透明度。

一句话总结：本轮重点不是复制更多协议能力，而是把 Risk Alert、Risk Value、sTRX 和 Energy Rental 串成更顺手的闭环：先解决看不懂、做不稳、失败难恢复，再谈重协议扩张。

5未来拓展

这一节不展开研究细节，只保留三件事：机会是什么、现在能不能做、当前先准备什么。
协议层能力储备
结论：这几项都有业务价值，但都需要协议层窗口，当前只做方案储备，不进入本轮排期。
中长期储备 协议升级后

E-Mode 效率模式（参考 Aave V3 / Venus）

  • 机会：提升 TRX/sTRX、USDT/USDD 这类高相关资产对的借款空间，是明显的 TVL 杠杆。
  • 当前结论：需要协议层支持资产分组和独立清算参数，本轮不纳入。
  • 现在先做：先准备资产分组方案和治理讨论材料，等合约升级窗口再推进。
长期观察 新风险池

信用贷 / 链上无抵押借贷（参考 Lista Credit）

  • 机会：若 TRON 链上信用体系成熟，可以显著扩大无大额抵押用户的借款覆盖面。
  • 当前结论：这不是前端优化，而是全新的风险池和信用体系建设。
  • 现在先做：持续观察链上身份与信用基础设施，并积累用户历史行为数据口径。
中长期储备 机构需求

固定利率 / 固定期限借款（参考 Lista DAO）

  • 机会：对机构和策略用户有明显吸引力，也是和浮动利率协议拉开差异的关键能力。
  • 当前结论：前端不难，真正的门槛在于协议能否支持利率锁定。
  • 现在先做：先确认合约可行性；在此之前，可用“利率区间预警”作为过渡表达。
产品形态演进
结论：先做轻量验证，再决定是否做重产品，避免一下子把团队拖进高投入、长周期项目。
中长期储备 自动化

Position Manager 策略自动化（参考 Aave Pro V4）

  • 机会：自动化再平衡对高净值和机构用户很有价值，是从“工具平台”走向“策略平台”的一步。
  • 当前结论：需要授权执行机制和自动化基础设施支撑，当前条件还不成熟。
  • 现在先做：先做 watch-only 地址簿和多地址管理体验，作为后续自动化的铺垫。
中期验证 先验证网页端

消费级移动 App（参考 Aave App）

  • 机会：TRON 用户天然偏移动端，消费级 App 有望带来非专业用户增量。
  • 当前结论：完整 App 投入太大，不适合在核心闭环还没打顺时启动。
  • 现在先做：先在网页端验证收益规划器和秒级计息，再评估 PWA 或 Mini App。
基础设施依赖
结论：这几项价值都不低，但更依赖 TRON 生态和外部条件成熟，当前只能持续跟踪。
基础设施成熟后 合规依赖

机构独立界面（参考 Spark Prime）

  • 机会：能把现有大额用户正式产品化，并进一步吸引机构资金进入。
  • 当前结论：真正的机构界面离不开 KYC/AML 和托管合作，不是单纯前端项目。
  • 现在先做：先做机构看板和多地址风险聚合，为后续机构入口打底。
生态依赖 AA / Paymaster

意图型交易 / Gasless 操作（参考 Aave Pro CoW Protocol）

  • 机会：如果未来能做到无 Energy 操作，会显著降低首次交易门槛。
  • 当前结论：高度依赖 TRON 的账户抽象、Paymaster 或意图求解器成熟。
  • 现在先做：先把 Energy 预估和一键租用做顺，再持续跟踪底层基础设施进展。
长期观察 风险框架

Morpho Blue 无许可市场创建

  • 机会：如果 TRON 长尾资产不断增加，无许可市场能带来更快的资产扩张和生态飞轮。
  • 当前结论：这会明显放大预言机、流动性和风控挑战，现阶段不宜贸然开放。
  • 现在先做：先把现有市场参数展示做透明，后续再考虑“待验证市场”沙盒机制。
